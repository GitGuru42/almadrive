# Анализ проекта AvtoRend

## Обзор проекта
AvtoRend — это веб-приложение, предоставляющее API для аренды автомобилей и обслуживающее свой собственный фронтенд. Проект реализован на Python с использованием фреймворка FastAPI. Для работы с базой данных используется SQLAlchemy. Проект поддерживает развертывание в производственной среде с использованием Docker и Render.

## Основные технологии:
*   **Бэкенд:** Python, FastAPI
*   **База данных:** PostgreSQL (рекомендуется), SQLAlchemy (ORM), Alembic (миграции)
*   **Статические файлы (Фронтенд):** HTML, CSS, JavaScript (обслуживается FastAPI)
*   **Хранение файлов:** Cloudinary
*   **Мессенджер:** Telegram (через python-telegram-bot)
*   **Развертывание:** Docker, Render, Gunicorn (для продакшена)

## Структура проекта
Проект имеет следующую структуру:
*   `api/`: Содержит логику бэкенда, включая маршруты FastAPI, модели SQLAlchemy, схемы Pydantic и утилиты для работы с базой данных, Cloudinary и Telegram.
*   `js/`: Каталог для JavaScript файлов фронтенда.
*   `locales/`: Каталог для файлов локализации.
*   `static/`: Каталог для статических файлов, таких как изображения, favicon.
*   `styles/`: Каталог для CSS файлов.
*   `index.html`: Главный HTML файл фронтенда.
*   `sw.js`: Файл Service Worker.
*   `Dockerfile`: Определение Docker-образа для приложения.
*   `PRODUCTION_NOTES.md`: Документация по развертыванию в продуктивной среде.
*   `render.yaml`: Конфигурация для развертывания на платформе Render.
*   `requirements.txt`: Список зависимостей Python проекта.
*   `start-dev.sh`: Скрипт для запуска проекта в режиме разработки.
*   `start.sh`: Скрипт для запуска проекта в продуктивном режиме.

## Сборка и запуск

### Зависимости:
Все зависимости Python указаны в `requirements.txt`.

### Локальный запуск (режим разработки):
1.  Установите зависимости:
    ```bash
    pip install -r requirements.txt
    ```
2.  Установите переменные окружения (пример, возможно потребуется настройка `DATABASE_URL`):
    ```bash
    export ENVIRONMENT=development
    export DATABASE_URL="sqlite:///./sql_app.db" # Пример для SQLite
    ```
3.  Запустите приложение:
    ```bash
    ./start-dev.sh
    ```

### Локальный запуск (продуктивный режим):
1.  Установите зависимости:
    ```bash
    pip install -r requirements.txt
    ```
2.  Установите переменные окружения:
    ```bash
    export ENVIRONMENT=production
    export DATABASE_URL="postgresql://user:password@host:port/database"
    # Дополнительно: export ALLOWED_ORIGINS="http://localhost:8000"
    ```
3.  Запустите приложение:
    ```bash
    ./start.sh
    ```

### Запуск с Docker:
1.  Соберите Docker-образ:
    ```bash
    docker build -t avtorend .
    ```
2.  Запустите Docker-контейнер:
    ```bash
    docker run -p 8000:8000 \
      -e ENVIRONMENT=production \
      -e DATABASE_URL="postgresql://user:password@host:port/database" \
      avtorend
    ```

### Развертывание на Render:
Проект включает файл `render.yaml` для легкого развертывания на Render. Необходимо установить переменные окружения `DATABASE_URL` и, опционально, `ALLOWED_ORIGINS` на панели управления Render.

## Правила разработки
*   **Обработка ошибок:** Приложение обрабатывает исключения HTTP и общие исключения, возвращая JSON-ответы. В режиме отладки предоставляются подробности трассировки стека.
*   **CORS:** Приложение настроено для обработки CORS-заголовков. Разрешенные источники задаются через переменную окружения `ALLOWED_ORIGINS`.
*   **Миграции базы данных:** Приложение не выполняет автоматическое создание схем или начальное заполнение данных при запуске. Используются миграции Alembic (подразумевается, что они запускаются отдельно).
*   **Переменные окружения:** Важные конфигурации (например, `DATABASE_URL`, `ENVIRONMENT`, `ALLOWED_ORIGINS`) загружаются из переменных окружения.
*   **Логирование запросов:** Приложение включает промежуточное ПО для логирования HTTP-запросов, которое может быть отключено в производственной среде.
